## üìù ISSUE REPORTING

When you find an issue:

1. **Document the issue:**
   - Issue title
   - Severity (Critical/High/Medium/Low)
   - Page/Route affected
   - Steps to reproduce
   - Expected vs actual result
   - Browser and OS
   - Screenshot (if applicable)

2. **Report the issue:**
   - Use issue reporting template in testing guides
   - Include all details above
   - Assign to appropriate team member
   - Set priority level

3. **Track the issue:**
   - Monitor resolution
   - Re-test when fixed
   - Verify fix works
   - Close issue
