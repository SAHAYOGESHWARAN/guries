## Workflow Examples

### Example 1: Creating a WEB Asset
1. User opens upload form
2. Content Type is already set to "WEB (Default)"
3. User fills in other fields
4. User can change content type if needed
5. User saves or submits

### Example 2: Creating an SEO Asset
1. User opens upload form
2. User changes Content Type from WEB to "SEO"
3. **Content Type immediately locks** ğŸ”’
4. User fills in other fields
5. User cannot change content type anymore
6. User saves or submits

### Example 3: Creating an SMM Asset
1. User opens upload form
2. User changes Content Type from WEB to "SMM"
3. **Content Type immediately locks** ğŸ”’
4. User fills in other fields
5. User cannot change content type anymore
6. User saves or submits

### Example 4: Editing an Existing Asset
1. User clicks edit on an existing asset
2. If asset type is SEO or SMM, field shows locked state
3. If asset type is WEB, field remains editable
4. User can modify other fields
5. User saves changes
